## 🙋‍♀️ ARC가 뭔가요?

> *이력은 단순한 스펙 목록이 아니다.*
> *어떤 동기로 그 활동을 선택했는지, 어떤 고민을 했는지, 무엇을 배웠는지 — 그 정성적 경험의 총체가 진짜 이력이다.*

**ARC**는 **Story Arc**에서 착안한 이름입니다. 소설과 영화처럼 인생에도 내러티브의 흐름이 있고, ARC는 그 흐름을 함께 쌓아가는 플랫폼입니다. **Archive**의 약어이기도 하죠.

대학생활 동안 수많은 활동을 했지만, 노션에 뿔뿔이 흩어진 기록들을 하나의 스토리로 엮는 것이 얼마나 어려운지 알기 때문에 만들었습니다. ARC는 그 허들을 없애고, 당신이 미처 몰랐던 자신의 스토리라인을 발견하게 해줍니다.

**우리가 풀려는 문제:**
- 📦 활동은 많은데 어디에 어떻게 정리해야 할지 모르겠다
- 🔗 수업·활동·인턴이 서로 연결이 안 되는 것 같아 불안하다
- 📝 자소서 쓸 때마다 "내가 뭘 했지?"부터 다시 시작한다
- 😰 뭐라도 해야 할 것 같은 강박, 앞으로 뭐해먹고 살지에 대한 불안

**ARC가 제공하는 것:**
- ✅ 직관적인 UI로 모든 경력·활동을 **쉽게 기록**
- 🤖 AI 챗봇이 대화를 통해 **파편화된 경험을 복기**하고 자동 정리
- 🔍 활동 간 **논리적 연결고리**를 찾아주는 내러티브 엔진
- 📄 아카이브 기반 **이력서·자소서 초안** 자동 생성

## 🌈 같이 만들어요

ARC는 현재 **대회용 Phase 1 MVP**를 개발 중입니다. 기여하고 싶다면:

```bash
# 1. 레포 포크
git fork https://github.com/arc-platform/arc-frontend

# 2. 브랜치 생성
git checkout -b feat/your-feature-name

# 3. 작업 후 PR 생성
# PR 제목: [feat] 기능명 / [fix] 수정내용 / [docs] 문서
```

**우선순위 원칙:**
1. 🥇 **기록의 허들을 낮추는 것이 최우선** — 사용자가 기록을 많이 쌓을수록 분석 품질이 올라가는 구조이므로, UX는 항상 입력 편의성 중심으로
2. 🥈 Phase 1 완성도에 집중 — Phase 1.5는 방향성 전달에 집중
3. 🥉 정성적 데이터 수집 품질 — AI 분석의 인프라는 기록 단계에서 결정됨

**기여 가능 영역:**
| 영역 | 내용 |
|------|------|
| 🎨 Frontend | React 기반 UI/UX, 대시보드, 타임라인 |
| ⚙️ Backend | API 설계, 데이터 모델링 |
| 🤖 AI/ML | 챗봇 경험 추출, 내러티브 엔진, NLP 태깅 |
| 📐 Design | Figma 컴포넌트, 브랜딩, CI |
| 📋 Planning | 기획 문서, 사용자 리서치 |

## 🍿 팀 소개

우리 팀은 "내 대학생활 기록을 한번에 정리할 플랫폼이 있으면 좋겠다"는 한 마디에서 시작했습니다.

> *"멋들어지는 자소서를 미리 써놓고 그 스토리라인에 맞춰서 스펙을 준비한다"*
